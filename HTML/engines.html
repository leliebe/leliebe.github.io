Game Engines

Low Code/No Code Engines

Bitsy

Description: From the developer: "bitsy is a little editor for little games or worlds. the goal is to make it easy to make games where you can walk around and talk to people and be somewhere." Bitsy is an entirely web-based engine.

Languages: Bitsy builds HTML files, but no knowledge of HTML is necessary for most purposes.

What it's good for: Bitsy works well for simple, story-focused games with a limited color palette and minimal, variable-focused mechanics. The "What you see is what you get" web-based development interface is good for beginner developers.

What it doesn't do well: By default, Bitsy is very simple. Any mechanics beyond conversation and single variable-based mechanics (usually inventory-focused) will require either using the engine in creative ways or seeking out additional tools.

Twine

Description: Twine is an HTML-based engine for creating interactive fiction.

Languages: Twine games exist as HTML files. Knowledge of CSS and JavaScript allow for more complex games.

What it's good for: Twine excels at branching narrative games. It can handle complex variable systems and conditional logic, as well as some multimedia integration. Twine has both a web-based interface and a desktop application.

What it doesn't do well: Twine's focus on interactive stories means that it is not sutied for text-light games. Depending on which of the four built-in formats being used, multimedia integration may be complicated. Mechanically, most player-input must be done via hyperlinks.

  • Additional Resources:
  • Twine Reference Guide
  • The Twine Cookbook: Documentation and Examples
  • Story Format: Chapbook Documentation Chapbook uses a custom format of "inserts" and "modifiers" rather than macros (used in Harlowe and SugarCube) or JavaScript libraries (used in Snowman).
  • Story Format: Harlowe Documentation Harlowe is the default story format for Twine.
  • Story Format: Snowman Documentation Snowman is designed for those with knowledge of JavaScript libraries.
  • Story Format: SugarCube Documentation SugarCube is similar to Harlowe, but requires more experience with programming methodologies.

Lua Engines

LÖVE

Description: From the developer: "LÖVE is an *awesome* framework you can use to make 2D games in Lua. It's free, open-source, and works on Windows, macOS, Linux, Android and iOS."

Languages: Python

What it's good for: Making 2D games in a scripting language with a fairly gentle learning curve. LÖVE is also open source, free to use, and has a fairly active development community.

What it doesn't do well: 3D games.

  • Additional Resources:
  • LÖVE Documentation

Python Engines

Ren'Py

Description: From the developer: "Ren'Py is a visual novel engine – used by thousands of creators from around the world – that helps you use words, images, and sounds to tell interactive stories that run on computers and mobile devices. These can be both visual novels and life simulation games. The easy to learn script language allows anyone to efficiently write large visual novels, while its Python scripting is enough for complex simulation games."

Languages: Python

What it's good for: Making visual novels

What it doesn't do well: Anything other than visual novels.
